# Recording

First feature beyond the updater. Captures audio from the default microphone
into a WAV file in app private storage; displays the active mic name.

## What ships in this version

- Start/stop recording button on main screen.
- Display of currently active mic name (from `AudioRecord.routedDevice`).
- Display of elapsed recording time (mm:ss.t).
- WAV file written to `filesDir/recordings/bird_<timestamp>.wav`.
- Display of last saved file name and size in KB.
- Permission request for `RECORD_AUDIO` on first start.

## What was deliberately not included

- F-tag dev overlays (`tagF1`, `tagF7`, `tagF8` from legacy). Tagging system
  retired; if a debug overlay returns later, it will use memory IDs (`n201`,
  `n202`, ...), not the F-scheme.
- Overflow menu, delete-all action — deferred. May return as separate nodes.
- dBFS level meter (legacy F4) — was dropped on legacy main; not reintroduced.

## Implementation

In `MainActivity.kt`, code lines tied to this leaf are marked `// n201`.
Key functions: `startRecording`, `stopRecording`, `writePcmToWav`,
`buildWavHeader`, `micDisplayName`, `requestMicAndStart`, `formatElapsed`.

## Status

Built and verified on device. APK signed by the same keystore as the
previous build, so updater handled the upgrade in place.
