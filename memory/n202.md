# Spectrum + dBFS stats

Live frequency-domain visualization and signal statistics, computed during
recording (n201).

## What ships in this version

- Live FFT spectrum view (`SpectrumView`), updated as the recording loop
  fills a 2048-sample window.
- Max and min peak amplitudes converted to dBFS, displayed above the spectrum.
- All stats reset when a new recording starts.

## What was deliberately not included

- Persistence of spectra across recordings.
- Configurable FFT parameters (window size, hop). Uses `FFT_SIZE = 2048`
  inherited from legacy.
- Spectrogram with a time axis. Only the current frame is shown.

## Implementation

`FftAnalyzer.kt` and `SpectrumView.kt` brought from `legacy` unchanged.
In `MainActivity.kt`, code lines tied to this leaf are marked `// n202`.
Stats live in `maxPeakSample` / `minPeakSample` (`AtomicInteger`),
updated inside the recording loop.

## Depends on

n201 (Recording) — analysis runs in the recording loop.

## Status

Built and verified on device.
