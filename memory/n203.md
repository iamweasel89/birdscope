# Phase portrait

Visualizes the audio signal as a 2D plot using delay coordinates:
each point is `(s[t], s[t+tau])` for a fixed delay `tau`.
Shows dynamical structure that the spectrum does not.

## What ships in this version

- `PhasePortraitView` — custom View; connected line through consecutive
  `(s[t], s[t+tau])` points.
- Window size: 1024... actually 4096 samples per repaint (~93 ms at 44.1 kHz,
  ~11 frames per second).
- Decimation step 4: every 4th sample drawn, not every sample.
- tau = 32 samples (~720 microseconds at 44.1 kHz). Compensates decimation,
  keeps geometric form similar to earlier tau=8 + step=1 attempts.
- Auto-scale: smoothed peak amplitude across windows (EMA, factor 0.85),
  peak floor 500 to avoid stretching pure noise. Signal occupies ~90% of
  the available radius regardless of input level.
- Toggle button below the visualization area switches between spectrum
  (`SpectrumView`) and phase portrait (`PhasePortraitView`). Stacked in
  a `FrameLayout`; only one visible at a time.
- Dark background `#0a0a0a`, green line `#7CFC00`, axes `#333333`.
  Visual style matches `SpectrumView`.

## What was deliberately not included

- Configurable tau, window, decimation. Fixed for v1.
- Fading or accumulation across windows. Each window replaces the previous.
- 3D phase portraits.
- Per-sample point cloud or color gradients.
- Adaptive peak floor. Floor 500 is a compromise: tight pure-tone fits
  fill the canvas; near-silence shows mild noise structure rather than
  going to a single dot.

## History of tuning

- v1 (initial): window 1024, tau=8, step=1, scale fixed to int16 range.
  Result: tiny dot in center for normal signals.
- v2: added auto-scale with EMA, peak floor 1500. Better, but still flickered
  and looked jagged.
- v3: window 4096, tau=32, step=4. Less flicker, much smoother line.
- v4 (current): peak floor lowered to 500. Quieter signals fill more of
  the canvas; near-silence remains acceptable.

## Implementation

In `MainActivity.kt`, code lines tied to this leaf are marked `// n203`.
Phase samples accumulate in `phaseAccum: ShortArray(4096)`; on full window,
`PhasePortraitView.setSamples(phaseAccum)` is called and the view repaints.
View toggling lives in `applyVisualizationMode()` and the click handler on
`togglePhaseButton`.
